Core Features

🎯

Smart Routing

Intelligent request distribution based on priority and health, with smooth weighted round-robin traffic balancing

🔄

Auto Failover

Switch away from failed channels in seconds, with exponential cooldown to avoid cascading failures

📊

Real-time Monitoring

Detailed request statistics, cost analysis, trend charts and a real-time request monitoring dashboard

💰

Cost Control

Daily channel cost limits and API token cost limits, tracked with micro-dollar precision

🔌

Multi-API Support

Fully compatible with Claude, Codex, Gemini and OpenAI clients, using one shared gateway configuration

⚡

Local Token Count

<5ms response time, 93%+ accuracy, token counting without upstream API calls

🔀

Protocol Transform

Transform between Anthropic, OpenAI, Gemini and Codex while preserving sampling, thinking and built-in search intent

🛡️

Soft Error Detection

Detect errors disguised as HTTP 200 responses and rate-limit markers in SSE streams

🌐

Per-channel Proxy

Route individual channels through HTTP, HTTPS, SOCKS5 or SOCKS5H proxies without changing global proxy policy

📍

DNS Host Override

Pin broken upstream hostnames to fixed IPs while preserving TLS SNI, certificates and Host headers

⏳

Quota-aware Cooldown

Understand provider fixed-window quota responses and cool down the channel until the real reset time

Not a black-box proxy

ccLoad puts channels, models, tokens, cost, first-byte latency, failure reasons, and chat-style model testing into one console. When routing looks wrong, read the data; when upstreams fail, read the logs.

  • Track requests, tokens, cost and latency by channel, model and token.
  • Run chat-style model testing with image upload, reasoning level, built-in search and streaming controls.
  • Configure multiple URLs, multiple keys, RPM limits, concurrency limits and daily cost limits per channel.
  • Export conversations as Markdown or HTML, then inspect masked upstream requests and responses when tracking protocol transform issues.

Quick Start

bash 
# Download docker-compose.yml and the .env template
curl -o docker-compose.yml https://raw.githubusercontent.com/caidaoli/ccLoad/master/docker-compose.yml
curl -o .env https://raw.githubusercontent.com/caidaoli/ccLoad/master/.env.example

# Edit .env and set the admin password
nano .env  # set CCLOAD_PASS=your_password

# Start the service
docker-compose up -d

# Open the admin console
# http://localhost:8080/web/
steps 
1. Sign in to Hugging Face: https://huggingface.co
2. Create a new Space: New Space -> Docker -> SDK: Docker
3. Create Dockerfile:
   FROM ghcr.io/caidaoli/ccload:latest
4. Configure environment variables (Settings -> Variables):
   CCLOAD_PASS=your_admin_password
5. Wait for the build to finish (about 2-3 minutes)
6. Open your Space URL
bash 
# Clone the repository
git clone https://github.com/caidaoli/ccLoad.git
cd ccLoad

# Set environment variables
export CCLOAD_PASS=your_admin_password

# Build (must use -tags sonic)
make build

# Run
./ccload

# Open http://localhost:8080/web/
