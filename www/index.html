不是黑盒代理

ccLoad 把渠道、模型、令牌、成本、首字延迟和失败原因放到同一个后台里。路由策略出了问题，看数据；上游抽风，看日志。

  • 按渠道、模型、令牌统计请求量、Token、成本和延迟。
  • 实时日志区分成功、失败、限流、首字超时和流式中断。
  • 渠道支持多 URL、多 Key、RPM 限制、并发限制和每日成本限额。
  • 调试日志可查看脱敏后的上游请求与响应，定位协议转换问题。

快速开始

bash 
# 下载 docker-compose.yml 和 .env 配置文件
curl -o docker-compose.yml https://raw.githubusercontent.com/caidaoli/ccLoad/master/docker-compose.yml
curl -o .env https://raw.githubusercontent.com/caidaoli/ccLoad/master/.env.example

# 编辑 .env 设置管理密码
nano .env  # 设置 CCLOAD_PASS=your_password

# 启动服务
docker-compose up -d

# 访问管理界面
# http://localhost:8080/web/
步骤 
1. 登录 Hugging Face: https://huggingface.co
2. 创建新 Space: New Space -> Docker -> SDK: Docker
3. 创建 Dockerfile:
   FROM ghcr.io/caidaoli/ccload:latest
4. 配置环境变量 (Settings -> Variables):
   CCLOAD_PASS=your_admin_password
5. 等待构建完成（约 2-3 分钟）
6. 访问你的 Space URL
bash 
# 克隆仓库
git clone https://github.com/caidaoli/ccLoad.git
cd ccLoad

# 设置环境变量
export CCLOAD_PASS=your_admin_password

# 构建（必须 -tags sonic）
make build

# 运行
./ccload

# 访问 http://localhost:8080/web/
bash 
# 前往 GitHub Releases 页面下载对应平台的二进制文件
