启动环境变量

启动项解决的是“服务能不能安全跑起来”。运行后可热更新的系统策略，不应该塞回环境变量里。

变量 默认值 用途 建议
CCLOAD_PASS 无 管理后台登录密码，未设置会直接退出。 生产必填，使用平台 Secret 或 Docker Secret。
CCLOAD_API_TOKENS 无 启动时预置 API 访问令牌，支持 token|描述。 适合自动恢复；常规管理优先用后台令牌页。
API_TOKENS 无 CCLOAD_API_TOKENS 的兼容别名。 不要和 CCLOAD_API_TOKENS 设置成不同值。
CCLOAD_MYSQL 无 MySQL DSN，设置后使用 MySQL，否则使用 SQLite。 多实例、Hugging Face 长期运行或外部备份时设置。
CCLOAD_ENABLE_SQLITE_REPLICA 0 混合存储开关：写 MySQL，读 SQLite 副本。 Hugging Face Spaces 推荐配合 MySQL 使用。
CCLOAD_SQLITE_LOG_DAYS 7 混合模式启动时从 MySQL 恢复日志的天数。 -1 全量，0 不恢复日志。
CCLOAD_ALLOW_INSECURE_TLS 0 允许跳过上游 TLS 证书校验。 只用于受控内网排障，生产不要开。
CCLOAD_MAX_CONCURRENCY 1000 全局最大并发代理请求数。 按机器资源和上游限制调低，不要无限放开。
CCLOAD_MAX_BODY_BYTES 10485760 请求体最大字节数，Images API 自动放宽到 20MB。 公网部署保持默认，避免大包拖垮服务。

存储模式

先按部署场景选存储，不要为了“看起来生产化”默认上 MySQL。单机 SQLite 更简单，也更少运维面。

模式 配置 适用场景 取舍
纯 SQLite 不设置 CCLOAD_MYSQL 本地开发、个人部署、单实例服务。 最简单；不适合多实例共享。
纯 MySQL 设置 CCLOAD_MYSQL 标准生产环境、外部备份、跨机器迁移。 持久化强；读写延迟取决于数据库。
混合模式 CCLOAD_MYSQL + CCLOAD_ENABLE_SQLITE_REPLICA=1 Hugging Face Spaces、MySQL 延迟高但需要持久化。 写主库、读本地缓存；复杂度更高。

渠道配置

渠道是 ccLoad 的核心数据结构：一个渠道可以有多个 Key、多个上游 URL、独立冷却、RPM/并发限制和协议转换策略。

基础字段

  • name：渠道名，必须可读，日志和统计都依赖它。
  • type：Anthropic、OpenAI、Gemini、Codex 等上游协议类型。
  • url：支持逗号分隔多 URL；末尾 # 表示精确上游 URL，不自动拼路径。
  • api_key：支持单 Key 或多 Key，系统在渠道内做 Key 级重试。
  • priority：优先级越高越先用，同优先级走平滑加权轮询。

保护字段

  • rpm_limit：渠道 60 秒滚动请求上限，0 表示无限制。
  • max_concurrency：渠道同时在飞请求上限，流式请求到结束才释放槽位。
  • daily_cost_limit：渠道每日成本限额，达到后跳过该渠道。
  • cost_multiplier：渠道倍率，按倍率后成本做限额统计。
  • scheduled_check_enabled：定时探活，配合检测模型使用。
多 URL 是同一渠道内部的容灾，不是渠道优先级替代品。系统会探索优先并按 1/EWMA 延迟加权选择，失败 URL 独立指数退避，不拖累同渠道其他 URL。

管理后台热配置

这些设置保存在数据库里，支持运行时调整。它们属于策略，不属于启动参数。

🔁

重试与冷却

Key 级错误在渠道内换 Key；5xx、598、599 等渠道级错误触发渠道切换。冷却使用指数退避，默认最高 30 分钟。

⏱️

协议超时

支持全局首字超时、非流式超时，也支持 Anthropic/Codex/OpenAI/Gemini 按上游协议覆盖。

📈

健康度排序

启用后按历史成功率动态调优有效优先级，并用置信度因子避免小样本渠道被过早打死。

💳

令牌限额

Auth Token 可配置允许模型、允许渠道、成本限额和统计范围；费用使用微美元整数避免浮点误差。

🧩

自定义请求规则

按渠道改写 Header 和 JSON Body，支持 remove/override/append；认证头被保护，禁止 CRLF 注入。

🧪

渠道检测

手动测试和定时检测都会计入渠道 RPM 限制。检测模型建议选择低成本、稳定可用的模型。

安全底线

代理服务的风险不是页面不好看，是把上游 Key 和管理入口暴露给不该访问的人。

上线前检查

  • 强密码设置在 Secret，不提交到仓库。
  • 至少创建一个 API 访问令牌，否则 `/v1/*` 会按设计返回 401。
  • 公网环境限制管理后台访问来源，或放到受保护网络后面。
  • 不要打开 `CCLOAD_ALLOW_INSECURE_TLS` 作为长期方案。

配置示例

.env 
CCLOAD_PASS=your_strong_admin_password
CCLOAD_API_TOKENS=client_token|production

# 可选：MySQL / 混合模式
CCLOAD_MYSQL=user:pass@tcp(host:3306)/ccload?charset=utf8mb4
CCLOAD_ENABLE_SQLITE_REPLICA=1
CCLOAD_SQLITE_LOG_DAYS=7
