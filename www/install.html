Docker 部署

这是生产环境最少踩坑的路径。先拉取 compose 和环境变量模板，只改密码，然后启动。

准备配置

复制官方 `docker-compose.yml` 和 `.env.example`，将 `.env` 里的 `CCLOAD_PASS` 改成强密码。

启动服务

默认监听 `8080`，管理后台在 `/web/`，API 代理在 `/v1/*` 和 `/v1beta/*`。

配置访问令牌

登录管理后台后进入令牌页创建 API token。没有 token 时代理接口会返回 401，这是安全默认值。

bash 
curl -o docker-compose.yml https://raw.githubusercontent.com/caidaoli/ccLoad/master/docker-compose.yml
curl -o .env https://raw.githubusercontent.com/caidaoli/ccLoad/master/.env.example

# 必填：设置管理后台密码
nano .env

docker-compose up -d
docker-compose logs -f ccload
MySQL 不是默认必需品。单机部署先用 SQLite。需要外部持久化、多实例或 Hugging Face 重启后保留数据，再设置 `CCLOAD_MYSQL`。

Hugging Face Spaces

Spaces 适合快速公开部署。根因风险是 `/tmp` 重启清空，所以长期使用要接外部 MySQL，并启用 SQLite 副本提升读取速度。

Space 设置

  • 新建 Space，SDK 选择 Docker。
  • `Dockerfile` 使用 `ghcr.io/caidaoli/ccload:latest`。
  • Secrets 添加 `CCLOAD_PASS`，不要写进公开文件。
  • 生产访问前，在管理后台创建 API 访问令牌。

持久化建议

  • 临时测试：只设 `CCLOAD_PASS`，使用默认 SQLite。
  • 长期使用：设置 `CCLOAD_MYSQL`。
  • 高频读取：额外设置 `CCLOAD_ENABLE_SQLITE_REPLICA=1`。
  • 日志恢复窗口用 `CCLOAD_SQLITE_LOG_DAYS` 控制。
Dockerfile 
FROM ghcr.io/caidaoli/ccload:latest

ENV PORT=7860
ENV SQLITE_PATH=/tmp/ccload.db
EXPOSE 7860

源码编译与二进制运行

源码路径用于开发和定制，二进制路径用于不想维护构建环境的轻量部署。两者都需要先设置 `CCLOAD_PASS`。

⚙️

源码编译

项目构建必须启用 `sonic` 标签。直接用 `make build`，不要手写一套分叉构建脚本。

bash 
git clone https://github.com/caidaoli/ccLoad.git
cd ccLoad

export CCLOAD_PASS=your_admin_password
make build
./ccload
📦

二进制运行

从 Releases 下载对应平台的产物，赋权后直接运行。版本固定比 `latest` 更容易回滚。

bash 
wget https://github.com/caidaoli/ccLoad/releases/download/v2.x.x/ccload-linux-amd64
chmod +x ccload-linux-amd64

export CCLOAD_PASS=your_admin_password
./ccload-linux-amd64

部署后验证

先验证进程活着，再验证配置入口，最后用真实 token 打代理请求。不要跳过 401，401 往往说明令牌还没配置。

bash 
curl -s http://localhost:8080/health
curl -s http://localhost:8080/public/summary

# 管理后台
# http://localhost:8080/web/

# API 代理端点示例
# POST http://localhost:8080/v1/messages
# POST http://localhost:8080/v1/chat/completions
# POST http://localhost:8080/v1beta/models/{model}:generateContent
