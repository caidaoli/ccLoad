API Usage

API Usage

ccLoad exposes Anthropic, OpenAI, Gemini, and Codex-compatible endpoints. Clients only need a new base URL and token.

Anthropic Messages OpenAI Chat Gemini v1beta Codex

Before calling the API

Create an API access token in the admin console, then point the client base URL at ccLoad. Do not send upstream provider keys to clients; clients should only hold ccLoad tokens.

Log in to the admin console

Open http://localhost:8080/web/ and log in with CCLOAD_PASS.

Add channels

Open the channel page and configure upstream type, URL, API key, priority, models and limits. Separate multiple URLs with commas.

Create access tokens

Create client tokens on the token page. Restrict models, choose a channel allowlist or denylist, and cap cost or concurrency when needed. An empty channel list is unrestricted in either mode. Proxy endpoints return 401 when no token exists.

Endpoint Reference

After protocol transforms are enabled, one upstream channel can serve multiple client protocols. The default upstream mode keeps native upstream semantics whenever possible.

Client Endpoint Purpose
Anthropic / Claude Code POST /v1/messages Messages API, streaming and non-streaming.
OpenAI-compatible clients POST /v1/chat/completions Chat Completions, suitable for OpenAI SDKs with only a base URL change.
Codex clients POST /v1/responses Responses-compatible endpoint for Codex-style requests and streams.
Codex Alpha Search POST /v1/alpha/search Native Codex search passthrough. Only resolved Codex upstreams are eligible.
Gemini clients POST /v1beta/models/{model}:generateContent Gemini generateContent-compatible endpoint.
Token count POST /v1/messages/count_tokens Local token estimation without forwarding to upstream.
Health check GET /health Process liveness check for container probes.

Built-in Search Across Protocols

ccLoad keeps model search requests in the native shape required by the runtime upstream protocol. It does not pretend web search is a normal function tool.

Protocol Search request shape Transform behavior
OpenAI Chat "web_search_options": {} OpenAI upstreams receive native Chat Completions search options; Codex web_search tools are converted to this shape when forwarding to OpenAI.
Codex Responses "tools": [{"type": "web_search"}] Codex upstreams receive the Responses tool form, including search options and tool choice when present.
Codex Alpha Search POST /v1/alpha/search Native passthrough only. This endpoint is never translated to another protocol, and its model field is optional.
Gemini "tools": [{"googleSearch": {}}] Gemini channels receive the Google Search tool shape instead of OpenAI or Codex fields.
Anthropic "web_search_20250305" Anthropic channels receive the Anthropic web search tool form with the standard tool name.
Search remains an upstream capability. If the selected provider, model or route does not support built-in search, disable search for that request or channel instead of expecting ccLoad to emulate browser access.

API Examples

The `Authorization` header below uses a ccLoad access token, not an upstream provider key.

bash 
curl -X POST http://localhost:8080/v1/messages \
  -H "Authorization: Bearer your_ccload_token" \
  -H "Content-Type: application/json" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "Hello from ccLoad"}
    ]
  }'
bash 
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Authorization: Bearer your_ccload_token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5",
    "messages": [
      {"role": "user", "content": "Search for current ccLoad release notes"}
    ],
    "web_search_options": {},
    "stream": true
  }'
bash 
curl -X POST http://localhost:8080/v1/responses \
  -H "Authorization: Bearer your_ccload_token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5-codex",
    "input": [
      {
        "type": "message",
        "role": "user",
        "content": [
          {"type": "input_text", "text": "Search for current ccLoad release notes"}
        ]
      }
    ],
    "tools": [{"type": "web_search"}],
    "tool_choice": "auto",
    "stream": true
  }'

Codex Alpha Search

This endpoint preserves the native Codex search payload. The model field is optional, and only channels whose resolved upstream protocol is Codex can handle the request.

bash 
curl -X POST http://localhost:8080/v1/alpha/search \
  -H "Authorization: Bearer your_ccload_token" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "golang channels"
  }'
Regular channel base URLs automatically append /v1/alpha/search. An exact URL ending in # must already point to that endpoint, for example https://upstream.example.com/v1/alpha/search#. ccLoad removes prompt_cache_key and prompt_cache_retention before forwarding.
bash 
curl -X POST "http://localhost:8080/v1beta/models/gemini-2.5-pro:generateContent" \
  -H "Authorization: Bearer your_ccload_token" \
  -H "Content-Type: application/json" \
  -d '{
    "contents": [
      {
        "role": "user",
        "parts": [{"text": "Hello from Gemini-compatible client"}]
      }
    ]
  }'
bash 
curl -X POST http://localhost:8080/v1/messages/count_tokens \
  -H "Authorization: Bearer your_ccload_token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-6",
    "messages": [
      {"role": "user", "content": "Count this locally"}
    ]
  }'

How to use the admin console

The console is not decorative; it is the runtime control plane for channel configuration, token limits, log tracing, trend statistics and debug logs.

  • Channels: create, edit, delete and test upstream channels; import or export CSV.
  • Tokens: assign client tokens with model, channel allowlist or denylist, cost and concurrency restrictions.
  • Logs: filter requests by status code, channel, model and token to locate failures.
  • Stats: inspect requests, tokens, cost, first-byte latency and health.
  • Trends: observe RPM, cost and latency by time range.

Channel Management API

The web UI is best for daily operations; bulk import, automation and migration can call the admin API directly.

Add a channel

bash 
curl -X POST http://localhost:8080/admin/channels \
  -H "Authorization: Bearer admin_session_or_token" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Claude-API",
    "type": "anthropic",
    "url": "https://api.anthropic.com",
    "api_key": "sk-ant-xxx",
    "priority": 10,
    "models": ["claude-sonnet-4-6"],
    "enabled": true
  }'

CSV import/export

bash 
curl -H "Authorization: Bearer your_token" \
  http://localhost:8080/admin/channels/export > channels.csv

curl -X POST -H "Authorization: Bearer your_token" \
  -F "file=@channels.csv" \
  http://localhost:8080/admin/channels/import

Debug from logs, not guesses

ccLoad records custom status codes such as 499, 596, 597, 598 and 599 to separate client cancellations, quota exhaustion, SSE errors, first-byte timeouts and stream interruptions.

429 has two explicit success-rate scopes. UI timeline success rate includes 429 as an error and exposes it again as the rate_limited subset for pure rate-limit coloring; health-score sorting excludes 429 so key-level throttling does not demote an otherwise healthy channel.
